ВИЖУФАСАД — обновление автоматизации v2

Что добавлено
1. Техническая проверка файла сохранена.
2. Добавлена необязательная смысловая AI-проверка фотографии дома.
3. Возможные статусы:
   queued_for_ai — заявка принята, AI ещё не подключён;
   queued_for_generation — фотография подходит, можно создавать варианты;
   photo_retake_required — клиенту нужно загрузить другое фото;
   photo_review_required — нужна ручная проверка.
4. Результат AI-проверки сохраняется в заказе и передаётся в MAX/Mail.ru.
5. При недоступности AI существующая отправка заявок продолжает работать.

Файлы из архива нужно загрузить в корень репозитория GitHub
с сохранением структуры папок и заменой существующих файлов.

После загрузки на GitHub выполнить на Timeweb по одной команде:

cd /opt/vizhufasad
git pull --ff-only
cd /opt/vizhufasad/server
npm ci --omit=dev
systemctl restart vizhufasad-api
curl -s https://89-23-97-248.sslip.io/health

Ожидаемый ответ до подключения AI-ключа:
{"ok":true,"service":"vizhufasad-leads","automation":"photo-ai-v2","ai":"not_configured"}

Важно
- Файл .env из сервера не удалять и не загружать в GitHub.
- Секретные ключи в GitHub и в чат не отправлять.
- OPENAI_API_KEY пока добавлять не требуется: сначала проверяем безопасное обновление.
