# Обработчик заявок ВИЖУФАСАД

Принимает форму с фотографией, создаёт постоянный номер заказа, проверяет техническое качество снимка, сохраняет заявку на сервере и параллельно отправляет её в закрытый чат MAX и на Mail.ru. При наличии ключа Yandex AI Studio дополнительно проверяет фотографию с помощью ИИ: видимость дома и фасада, читаемость геометрии, препятствия и пригодность ракурса.

Требуются Node.js 22+, HTTPS, верифицированный бот MAX и отдельный пароль приложения Mail.ru.

Скопируйте `.env.example` в `.env` и заполните значения только на сервере. `.env` запрещено добавлять в GitHub.

```bash
npm install
npm run check
npm start
```

Проверка: `GET /health` должна вернуть JSON с `"ok": true`.

ИИ-проверка необязательна. Без ключа сервис продолжает принимать и доставлять заявки, а статус остаётся `queued_for_ai`. С ключом пригодное фото получает статус `queued_for_generation`; явно непригодное — `photo_retake_required`; спорное — `photo_review_required`.

Для Timeweb рекомендуется `AI_PROVIDER=yandex`. Укажите `YANDEX_API_KEY`, `YANDEX_FOLDER_ID` и модель `YANDEX_MODEL` (по умолчанию `qwen3.6-35b-a3b`). Ключ должен иметь право выполнения моделей AI Studio. В режиме `AI_PROVIDER=auto` сервис сначала выбирает настроенный Yandex, а затем OpenAI. Текущий провайдер виден в безопасном ответе `/health`, сами секреты не выводятся.

Данные заказов сохраняются в каталоге `DATA_DIR` (по умолчанию `server/data`) и не попадают в GitHub. Статус заказа доступен по `GET /api/orders/:id/status?token=...`; секретный токен возвращается только клиенту при создании заказа.
